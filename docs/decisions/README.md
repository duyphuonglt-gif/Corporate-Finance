# docs/decisions/

Storage for important project decisions using Decision Record format (ADR).

## Purpose

Each file documents a single decision: the context, considered alternatives, the decision made, and its consequences. This helps the team understand *why* an approach was chosen, not just *what* was chosen.

## File Structure

```
# YYYY-MM-DD: <Decision Title>

## Context
## Alternatives Considered
## Decision
## Consequences
```

## Examples

- `2026-01-15_financial_modeling_tool_selection.md`
- `2026-02-10_valuation_methodology.md`
