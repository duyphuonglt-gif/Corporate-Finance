# BUS 629 VEMBA Corporate Finance
A comprehensive repository for financial modeling, valuation analysis, and strategic financial planning. This project develops and demonstrates enterprise-grade financial models, conducts rigorous analysis of investment opportunities, and delivers actionable insights for corporate decision-making.

---

## Overview

This repository contains a structured approach to corporate finance analysis, including discounted cash flow valuations, leveraged buyout structures, merger and acquisition modeling, and financial forecasting. All models follow best-practice conventions for transparency, reproducibility, and stakeholder communication.

## Creator
**Luong Duy Phuong** — Chief Technology Officer with 15+ years of expertise in enterprise architecture and financial technology integration. Brings a dual perspective of infrastructure scalability and data-driven financial decision-making to model design and implementation.

## Repository Structure

| Directory | Purpose |
|-----------|---------|
| `models/` | Financial model templates and completed builds (DCF, LBO, M&A, budgets, capex analysis) |
| `analysis/` | Validation, sensitivity analysis, scenario testing, and research findings |
| `deliverables/` | Finalized reports, presentations, and dashboards for stakeholder distribution |
| `data/` | Source data, assumptions, and reference materials |
| `docs/` | Documentation, specifications, decision records, and templates |

## Financial Modeling Areas

- **DCF Valuation** — Discounted cash flow analysis and enterprise value estimation
- **LBO Analysis** — Leveraged buyout structuring and return modeling
- **M&A Modeling** — Merger and acquisition valuation and deal scenarios
- **Financial Planning** — Budget preparation, cash flow forecasting, and scenario planning
- **Capital Expenditure** — CapEx analysis, ROI evaluation, and investment appraisal

## Conventions & Standards

### File Naming
- **Excel models**: `<model-type>_<project-name>_v<version>.xlsx`
- **Python scripts**: `<model-type>_<project-name>.py`
- **Analysis files**: `<analysis-type>_<topic>_<YYYYMMDD>.<ext>`
- **Deliverables**: `<type>_<project-name>_<audience>_<YYYYMMDD>.<ext>`

### Model Quality Standards
- Models maintain clear documentation of assumptions and drivers
- Sensitivity and scenario analysis validate key dependencies
- Version control tracks changes and maintains audit trails
- All deliverables undergo validation before stakeholder distribution
